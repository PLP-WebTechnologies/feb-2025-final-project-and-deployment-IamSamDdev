Understanding JavaScript Closures

Published on May 13, 2025

JavaScript closures are one of the most powerful features in the language, and also one of the most misunderstood. In simple terms, a closure is created when a function is defined inside another function, and the inner function has access to the outer functionâ€™s variables.

What is a Closure?

A closure is formed when an inner function remembers the environment in which it was created â€” even after the outer function has finished executing.


function outer() {
  let counter = 0;
  function inner() {
    counter++;
    console.log(counter);
  }
  return inner;
}

const increment = outer();
increment(); // 1
increment(); // 2

In the example above, the inner function retains access to the counter variable, even though the outer function has completed. Thatâ€™s because of closure.

Why Closures Matter

  • They allow data to be kept private (encapsulation)
  • Useful for function factories and currying
  • Powerful for callbacks and asynchronous code

Real-Life Use Cases

Closures are commonly used in event listeners, timer functions, and modules that need to maintain private state across function calls.

Conclusion

Closures can be tricky to grasp at first, but once you understand them, you'll have a much deeper understanding of how JavaScript works under the hood.
